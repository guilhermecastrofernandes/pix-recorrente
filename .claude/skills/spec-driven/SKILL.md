# Spec-Driven Development

Ao criar ou alterar funcionalidades:
1. Sempre leia os arquivos presentes no diretório `specs/`.
2. Valide se a alteração cumpre todos os critérios de aceite definidos no requisito.
3. Não implemente código sem que haja uma especificação clara ou um teste associado.
4. Caso encontre ambiguidade na especificação, pergunte antes de prosseguir.
5. Código deve ter expressões somente em Português como nomes de classe e métodos/funções.
6. Código em nenhum momento  deve ser commitado.
